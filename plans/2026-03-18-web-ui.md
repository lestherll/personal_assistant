## Vision

### 10x Check
A personal AI cockpit — not just a chat page. Open it and your most recent conversation is already there. ⌘K switches workspaces without touching the mouse. Code responses render with syntax highlighting and one-click copy. The usage dashboard maps your own cognition: which agents you use, what they cost, where you spend attention. Agent configuration is a form, not curl.

### Platonic Ideal
Open it and see your most recent conversation, streaming tokens already in flight. ⌘K to switch workspaces without the mouse. Code renders with syntax highlighting + copy button. Usage dashboard maps your cognition. Agent config is a form: name, system prompt, model picker, tool checklist. Instant, keyboard-navigable, dark-mode-aware.

## Design Review Addendum
Added by /plan-design-review on 2026-03-19 | Overall: 5/10 → 8/10

### Design System (Pass 5 — was 3/10)

**Fonts:** Replace `system-ui` with a proper web font. Use **Inter** (or Geist) via `@fontsource/inter` —
load weights 400, 500, 600. Apply globally via `body { font-family: 'Inter', system-ui, sans-serif; }`.

**Colour tokens** (document in DESIGN.md):
- Primary: `violet-600` (#7c3aed), hover: `violet-700`
- Destructive: `red-500`
- Success: `green-600`
- Surface: `bg-white dark:bg-gray-900`
- Background: `bg-gray-50 dark:bg-gray-950`
- Border: `border-gray-200 dark:border-gray-700`

**Radius:** `rounded-xl` cards, `rounded-lg` inputs/buttons, `rounded-2xl` message bubbles.

**DESIGN.md:** Create `DESIGN.md` at repo root documenting the above tokens, spacing scale (`p-4` cards,
`p-6` page padding, `gap-2`/`gap-3` forms), dark mode rules, and component vocabulary.

**3-way theme selector:** The sidebar exposes only a binary light/dark toggle but `ThemeContext` supports
`"system"` as a third value. Replace the toggle button with a 3-segment control: `Light | System | Dark`.
Default to `System` on first visit (already implemented in ThemeContext).

### Visual Identity (Pass 4 — was 4/10)

**App name + user avatar in sidebar header:** Replace the plain `"Personal Assistant"` label with:
- A coloured initial avatar derived from the logged-in username (e.g., first letter, background colour
  hashed from the username string using `charCodeAt` sum mod a palette of 6 violet/indigo/cyan/teal shades)
- Below the avatar: the username in `text-sm font-medium`

**Workspace avatars:** In both the sidebar nav and WorkspaceList cards, prefix each workspace entry
with a small circle avatar (24px) containing the workspace name's first letter. Colour derived from
workspace name hash (same palette as user avatar). This makes workspaces scannable when there are many.

### Information Architecture (Pass 1 — was 7/10)

**ConversationHistory responsive layout:** The left panel is `w-72 flex-shrink-0` which overflows on
mobile. On viewports `< lg`: hide the list panel by default, show a back button in the message preview.
Pattern: same slide-in approach as the main sidebar. Alternatively, stack the panels vertically on mobile
(list on top, preview below) with a `flex-col lg:flex-row` wrapper.

**Mobile header context:** The mobile header in `Layout.tsx` shows only `☰`. Add the current workspace
name (from URL param) as `text-sm font-medium` beside the hamburger so the user maintains spatial
awareness when the sidebar is closed.

**Resume last conversation:** After login, if the user has any existing conversations, redirect to the
most recent one instead of `/workspaces`. Implementation: in `AuthContext` after successful login,
call `GET /workspaces/` → take the first workspace → call `GET /workspaces/{name}/conversations?limit=1`
→ if a conversation exists, `navigate(/workspaces/{name}/chat/{id})`, else fall through to `/workspaces`.

### Interaction States (Pass 2 — was 4/10)

**UsageDashboard empty state:** When `summaryList` is an empty array (not undefined/loading), render:
```
No usage data yet.
Start a conversation with one of your agents to see token usage and cost analytics here.
```
Centred, `text-sm text-gray-400`, with a link to the default workspace chat.

**WorkspaceList empty state:** Replace `"No workspaces yet. Create one below."` with:
```
No workspaces yet.
A workspace groups agents and conversations around a topic or role.
[Create your first workspace ↓]   ← scrolls to create form
```

**ConversationHistory empty state:** Replace `"No conversations found."` with:
```
No conversations here yet.
Conversations appear after your first chat in this workspace.
[Go to chat →]   ← links to /workspaces/{name}/chat
```
Replace `"Select a conversation to preview"` with:
```
← Select a conversation on the left to read it here.
```

**ApiKeys empty state:** Replace `"No API keys yet."` with:
```
No API keys yet.
API keys let you access your assistant programmatically — from scripts, tools, or other apps.
```

### Accessibility (Pass 6 — was 4/10)

**aria-live for streaming chat:** In `Chat.tsx`, wrap the messages container with:
```tsx
<div aria-live="polite" aria-atomic="false" aria-relevant="additions">
  {messages.map(...)}
</div>
```
This announces new AI tokens to screen readers as they arrive without re-reading prior messages.

**Touch targets — 44px minimum:** Apply `min-h-[44px] min-w-[44px]` (or equivalent padding) to all
interactive elements that are currently undersized:
- WorkspaceList: "History" and "Delete" buttons (currently `text-xs px-2 py-1` ≈ 28px)
- ApiKeys: "Revoke" button
- Sidebar: `☰` hamburger button — add `p-2` so the tap area is adequate
- Sidebar nav links: current `py-1.5` is borderline; add `min-h-[36px]` as a floor

**Explicit form label associations:** In `Login.tsx`, `AgentConfig.tsx`, `WorkspaceList.tsx`,
`ApiKeys.tsx` — add `htmlFor` on every `<label>` and matching `id` on every `<input>` / `<textarea>`
/ `<select>`. Example: `<label htmlFor="agent-name">Name</label><input id="agent-name" .../>`.

### Confirmation UX (Pass 7)

**Replace `confirm()` with inline confirmation:** When the user clicks "Delete" (workspace) or
"Revoke" (API key), replace the button with an inline confirmation row:
```
[Cancel]  [Yes, delete]
```
This stays within the card, respects dark mode, and feels native to the design. State: add a
`confirming: string | null` state variable tracking which item ID is awaiting confirmation.
