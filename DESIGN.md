## Aesthetic

Clean, keyboard-first personal cockpit. Not a generic SaaS product — an opinionated tool for one user.
Dense information, low visual noise, violet identity, dark-mode-first.

## Components

### Avatar
Generated colour-coded initials circle. Size `sm` = 24×24px, `md` = 32×32px.
Colour derived from `charCodeAt` sum mod 6 across the avatar palette.
Used in: sidebar header (user), sidebar nav (workspaces), WorkspaceList cards.

### Message Bubble
- Human: right-aligned, `bg-violet-600`, white text, `rounded-2xl rounded-tr-sm`
- AI: left-aligned, `bg-gray-100 dark:bg-gray-800`, `rounded-2xl rounded-tl-sm`
- AI bubbles render Markdown + syntax-highlighted code via `react-markdown` + `rehype-highlight`
- Code blocks: `github-dark` highlight.js theme; copy button visible on hover

### Chat Input
- `textarea` auto-grows to 200px max
- Enter = send, Shift+Enter = newline
- Stop button (`bg-red-500`) replaces Send during streaming

### Command Palette
- Triggered by `⌘K` (Mac) / `Ctrl+K` (other)
- `cmdk` library — full keyboard navigation built-in
- Searches workspaces + static navigation targets

### Theme Selector
3-way segmented control: `Light | Auto | Dark`
- Located in sidebar footer
- Persisted to `localStorage` as `"light"`, `"system"`, or `"dark"`
- `"system"` follows `prefers-color-scheme` media query

## Dark Mode

Class-based (`darkMode: "class"` in `tailwind.config.ts`).
Applied to `<html>` via `ThemeContext` → `document.documentElement.classList`.
Custom variant: `@custom-variant dark (&:where(.dark, .dark *))` in `index.css`.

## Empty States

Every empty state must have: (a) a clear heading, (b) one sentence of context, (c) a primary action where applicable. Use `border-dashed border-gray-300` card with centred content.

| Screen | Heading | Context | CTA |
|---|---|---|---|
| WorkspaceList | "No workspaces yet" | "A workspace groups agents and conversations around a topic or role." | "Create your first workspace ↓" (scrolls to form) |
| ConversationHistory (list) | "No conversations here yet" | "Conversations appear after your first chat in this workspace." | "Go to chat →" |
| ConversationHistory (preview) | ← Select a conversation on the left to read it here | — | — |
| UsageDashboard | "No usage data yet" | "Start a conversation with one of your agents to see token usage and cost analytics here." | — |
| ApiKeys | "No API keys yet" | "API keys let you access your assistant programmatically — from scripts, tools, or other apps." | — |
| Chat (no agents) | — | "This workspace has no agents yet." | "Create an agent" button |

## AI Slop Avoidance

Things that make this design feel like *this product*, not a SaaS template:
- Inter font (not system-ui)
- Colour-coded workspace/user avatars (generated, no external images)
- Message bubble tails (`rounded-tr-sm` / `rounded-tl-sm`) on chat bubbles
- 3-way theme selector (not a binary toggle)
- Streaming cursor (inline pulse animation) during AI response
- ⌘K command palette as primary navigation affordance
- Usage dashboard as "cognitive map" framing (not just analytics)
